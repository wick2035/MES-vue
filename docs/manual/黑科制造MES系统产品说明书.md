# 第 1 章 产品概述

## 1.1 产品定位

黑科制造 MES（Manufacturing Execution System，制造执行系统）是一套面向离散制造企业的**生产现场管控平台**，定位于连接上层 ERP/订单与底层车间执行的中枢。系统围绕"**基础数据 → 产品与工艺设计 → 生产计划 → 派工下发 → 现场执行采集 → 库房物料管控 → 数据可视化**"这一主线，将基础数据、工艺、计划、物料、现场执行、审批流程与数据可视化统一在一个平台内，做到**过程清晰可视、数据可追溯**。

> 系统登录页标语：**"面向生产计划、工艺流转、物料追踪与现场执行的智能制造中枢。"**

## 1.2 核心价值

| 能力维度 | 说明 |
| :--- | :--- |
| 全链路数据贯通 | 物料 → 零部件 → BOM → 工艺路线 → 工序内容 → 生产订单 → 工单 → 派工 → 过站采集 → 出入库，环环相扣，单据可上下追溯。 |
| 三层 BOM 结构 | 支持"产品 BOM / 半成品 BOM / 组件 BOM"多层级嵌套，通过子 BOM 关联形成完整的物料清单树。 |
| 工艺规划与锁定 | 工艺流程支持按 BOM 初始化、草稿编辑、锁定定版，锁定后不可随意更改，保障现场执行依据稳定。 |
| 智能排产与派工 | 生产订单审核后自动 MRP 运算，并支持设备/员工两级派工，员工派工内置**负载均衡**算法。 |
| 闭环库房管理 | 申请—确认两段式作业，入出库均生成不可跳过的库存流水，支持手工/计划/配套等多种业务类型。 |
| 可配置审批流程 | 内置轻量工作流引擎（分类—模型—定义—实例—任务—办理），订单审批、工单变更审批等业务可挂接审批。 |
| 数据可视化 | 提供"智慧大屏"与"智能制造数据中心"两套大屏，以及 3D 数字孪生仓库。 |
| AI 智能辅助 | 接入通义千问大模型，提供"智能数据助手（自然语言查询）"与"AI 智能建模（产品→BOM→工艺→工单四步向导）"。 |

## 1.3 技术架构

| 层次 | 选型 |
| :--- | :--- |
| 运行平台 | Java 11 |
| 应用框架 | Spring Boot（单模块） |
| 持久层 | MyBatis-Plus + MySQL，Druid 连接池 |
| 缓存 | Redis、EhCache |
| 安全鉴权 | Apache Shiro（用户/角色/菜单权限） |
| 视图层 | FreeMarker（`.ftl` 模板）服务端渲染 |
| 前端 | layui + 自封装 `sp*` 组件（spTable / spLayer / spUtil） |
| 大模型 | 通义千问 DashScope（OpenAI 兼容接口，SSE 流式 + 工具调用） |

系统采用经典的**服务端渲染 + 异步取数**模式：页面框架由 FreeMarker 渲染，列表数据通过 `POST` 接口异步加载到自封装的 `spTable` 组件，编辑/选择类弹窗统一由 `spLayer` 承载。每个业务域按 `controller / entity / mapper / request / service` 分层组织。

# 第 2 章 运行环境与部署

## 2.1 运行环境要求

| 组件 | 要求 | 默认配置 |
| :--- | :--- | :--- |
| JDK | Java 11 | — |
| 数据库 | MySQL，库名 `sparchetype` | dev 连接 `127.0.0.1:3306` |
| 缓存 | Redis | `127.0.0.1:6379` |
| 服务端口 | 9090（无 context-path） | `server.port=9090` |
| 浏览器 | Chrome / Edge 等现代浏览器 | 推荐分辨率 ≥ 1440×900 |
| 上传目录 | 文件上传路径 | `D:/mes/upload`，访问前缀 `/upload` |

## 2.2 构建与启动

项目通过项目内的 Maven 配置文件 `.codex-maven-settings.xml`（阿里云镜像 + 项目内本地仓库）进行构建。常用命令（PowerShell）：

```powershell
# 启动应用（跳过测试）
mvn -s .\.codex-maven-settings.xml -f .\mes\pom.xml -DskipTests spring-boot:run

# 仅编译 / 打包
mvn -s .\.codex-maven-settings.xml -f .\mes\pom.xml -DskipTests compile
mvn -s .\.codex-maven-settings.xml -f .\mes\pom.xml -DskipTests package
```

启动成功后，浏览器访问 **http://localhost:9090** 即可进入登录页。默认激活 `dev` 配置（`application-dev.yml`）。

## 2.3 数据库脚本

系统**不使用** Flyway/Liquibase 自动迁移，所有结构变更与演示数据均以 SQL 脚本形式存放于 `scripts/sql/`，需**手动执行**。脚本均为可重复执行（`IF NOT EXISTS` / `INSERT IGNORE` / `NOT EXISTS` 子查询）。

> **提示**：执行 SQL 脚本导入中文数据时，须指定字符集 `--default-character-set=utf8mb4`，否则中文会乱码。完整演示数据可通过 `demo-data-full-reset-*.sql` 一键重建。

# 第 3 章 登录与主界面

## 3.1 登录系统

**访问路径**：`http://localhost:9090/login-ui`

![登录页](images/00-login.png)

**操作步骤**：

1. 在浏览器打开系统地址，自动进入登录页。
2. 输入**用户名/邮箱**（默认 `admin`）与**密码**（默认 `123`）。
3. 输入右侧**图形验证码**（4 位数字，点击图片可刷新）。
4. 如需保存登录状态，勾选"记住密码"。
5. 点击 **登 入** 按钮，校验通过后进入系统主页 `/admin`。

**字段与规则**：

| 字段 | 必填 | 说明 |
| :--- | :--- | :--- |
| 用户名/邮箱 | 是 | 支持用户名或邮箱登录 |
| 密码 | 是 | 凭证经 MD5（盐值为用户名，迭代 3 次）加密校验 |
| 图形验证码 | 是 | 4 位随机数字，服务端会话校验，错误将提示"请输入正确的验证码" |

> 登录异常会有明确提示：错误的帐号、密码错误、锁定的帐号、禁用的帐号、登录失败次数过多等。会话默认超时时间为 30 分钟（`session-timeout: 1800`）。

## 3.2 主界面与导航

登录后进入控制台主页，整体采用"**顶栏 + 左侧导航树 + 中部内容区（多标签页）**"布局。

![系统主页](images/01-admin-home.png)

**界面区域说明**：

| 区域 | 说明 |
| :--- | :--- |
| 顶部栏 | 左侧 Logo「智能 MES」，中部一级分组切换（常规管理 / OPC 操作 / 其他管理），右侧刷新、全屏、用户头像与"退出登录"。 |
| 左侧导航 | 按"业务中心"组织的多级菜单树，底部提供"全局搜索"快速定位菜单。 |
| 内容区 | 以多标签页（Tab）方式打开各功能页，欢迎页展示"核心模块"快捷入口与系统信息。 |

主页"核心模块"区提供四个快捷入口：**基础数据、工艺设计、生产工单、系统管理**，并在"系统信息"区展示系统名称、技术栈、前端框架与当前日期。

## 3.3 通用操作规范

系统各功能页高度统一，掌握以下通用规范即可快速上手所有模块。

### 3.3.1 查询区

页面顶部为查询条件区，输入/选择条件后点击 **查询** 过滤数据，点击 **重置** 清空条件。

### 3.3.2 数据表格（spTable）

- 列表数据分页加载，底部为分页条（上一页/下一页、跳页、每页条数）。
- 表格右上角通常提供"列筛选、导出、打印"等工具按钮。
- 列表常见状态以彩色标签呈现：绿色=正常/已完成、橙色=待处理/草稿、红色=异常/删除、蓝色=进行中。

### 3.3.3 增删改（CRUD）

| 操作 | 入口 | 说明 |
| :--- | :--- | :--- |
| 新增 | 工具栏「新增/添加」 | 弹出表单弹窗，填写后保存 |
| 编辑 | 行内「编辑/修改」 | 带入当前行数据的表单弹窗 |
| 删除 | 行内「删除」 | 二次确认后**软删除**（置 `is_deleted=1`），不物理删除 |
| 禁用/启用 | 行内「禁用/启用」 | 状态在 正常(0) ↔ 禁用(2) 间切换 |

### 3.3.4 主从联动

部分模块为"主表—从表"布局（如班组—员工、库房—库位、加工单元—班组、编组—设备）：**先在上方选中一条主表记录，下方从表才会刷新对应明细**。从表标题处会有"（请先在上方选择一个 XX）"的提示。

### 3.3.5 选择弹窗

新增/编辑表单中常见"选择"按钮（如选择物料、选择 BOM、选择设备、选择员工），点击后弹出选择子页，勾选后回传到主表单。

# 第 4 章 系统管理

系统管理面向平台管理员，负责菜单、用户、角色、部门等权限基础数据的维护，是整个系统的访问控制底座。

## 4.1 菜单管理

**路径**：常规管理 → 系统管理 → 菜单管理（`/admin/sys/menu/list-ui`）

![菜单管理](images/10-menu.png)

**功能用途**：以树形结构维护系统的全部导航菜单（目录/菜单/按钮），定义菜单名称、类型、图标、访问地址、权限标识与排序，是角色授权（授权菜单）的数据来源。

**界面说明**：以树表展示菜单层级，提供"全部展开/折叠"按钮；每行展示 名称、类型、图标、地址、权限标识、状态（锁定）及"修改/删除/添加（子菜单）"操作。

**操作流程**：

1. 点击行末 **添加** 为该节点新增子菜单；或点击顶部新增根级目录。
2. 在弹窗中填写：菜单名称、上级菜单、类型（目录/菜单/按钮）、路由地址、权限标识、图标、排序号。
3. 保存后菜单即出现在导航树（菜单数据存于 `sp_sys_menu` 表）。
4. 通过 **修改/删除** 维护已有菜单。

> 新增业务菜单通常通过 SQL 脚本 `INSERT IGNORE` 注册并授权给管理员角色（role code `888888`），以保证可重复部署。

# 第 5 章 基础数据中心

## 5.1 班组员工定义

**路径**：基础数据中心 → 班组员工定义（`/basedata/team/list-ui`）

## 5.2 编组设备定义

## 5.3 加工单元定义

![加工单元定义](images/22-processing-unit.png)

**功能用途**：定义生产的最小执行单位"加工单元"（如"电脑组装单元""整机测试单元""包装入库单元"），并为其绑定执行班组。加工单元是工序的承载者——工序信息定义中每道工序都归属于某个加工单元。

**字段说明**：

| 字段 | 说明 |

| 单元类型 | 人员作业单元 / 设备作业单元 |

**操作流程**：

1. 「添加」加工单元，填写编号、名称、标准产能、单元类型、是否有线边库。
2. 选中加工单元行，在下方"加工单元班组管理"为其绑定执行班组。

## 5.4 设备管理

**路径**：基础数据中心 → 设备（`/basedata/equipment/list-ui`）

![设备管理](images/23-equipment.png)

**功能用途**：维护车间设备/工装台账（设备编号、名称、状态、备注等），是"编组设备定义"与"设备作业派工"的设备来源。

**操作流程**：通过「新增/编辑/删除」维护设备主数据；设备可被加入编组、并在工艺-设备关联与设备派工中被引用。

## 5.5 库房库位定义

**路径**：基础数据中心 → 库房库位定义（`/basedata/warehouse/list-ui`）

![库房库位定义](images/24-warehouse.png)

**功能用途**：以"库房（主表）—库位（从表）"维护仓储空间结构。库房按**组×排×层×列**四维定义容量，系统据此**自动批量生成库位编码**（如 `KF001-1-1-1-1`）。

**字段说明**：

| 字段 | 说明 |
| :--- | :--- |
| 库房编码 / 名称 | 库房唯一编码与名称（如 CP001 产品库、KF001 配件电脑） |
| 库房类型 | 成品库 / 原材料库 等 |
| 组 / 排 / 层 / 列 | 库房的库位维度，决定自动生成的库位数量与编码 |
| 状态 | 正常 / 禁用 |

**操作流程**：

1. 「新增」库房，填写编码、名称、类型，并设置 组/排/层/列。
2. 保存后系统按维度自动生成库位。
3. 选中库房行，下方"库位管理"展示该库房下的全部库位（库位编码、所属库房、状态）。

## 5.6 库存管理

**路径**：基础数据中心 → 库存管理（`/basedata/inventory/list-ui`）

![库存管理](images/25-inventory.png)

**功能用途**：以"库位 + 物料 + 数量"为粒度维护当前库存账，是出入库业务扣减/登账的对象。每条库存记录精确到 库房-库位-物料编码-批号-数量-单位-库存状态。

**操作流程**：

1. 可按库房、物料、库位编码查询库存。
2. 点击「入库」可手工录入一条库存（指定库房、库位、物料、批号、数量）。
3. 行内「编辑/删除」用于盘点调整。

> 实际生产中，库存的增减应主要由"库房管理中心"的入出库确认驱动，以保证有完整流水可追溯；此处的直接维护通常用于初始化与盘点。

**字段说明**：零部件编码、所属产品名称、零部件名称、类型（组件/半成品）、状态、备注。

**操作流程**：「新增」零部件并指定其所属产品与类型；可按产品名称、零部件编码/名称、类型查询。

## 6.3 产品 BOM 管理

**路径**：产品数据中心 → 产品 BOM 管理（`/technology/bom/list-ui`）

![产品 BOM 管理](images/32-bom.png)

**功能用途**：维护产品的物料清单（Bill of Materials），系统支持**三层 BOM 层级结构**：

- **产品 BOM**：成品级清单（如 BOM-PC-HOST 台式电脑主机）；
- **半成品 BOM**：半成品级清单（如 BOM-PC-HOST-HALF 台式电脑主机半成品）；
- **组件 BOM**：组件级清单（如 BOM-MAINBOARD-UNIT 主板单元、BOM-CASE-UNIT 机箱单元）。

上层 BOM 行通过 `child_bom_id` 关联下层子 BOM，从而形成可逐层展开的 BOM 树。

**字段说明**：

| 字段 | 说明 |
| :--- | :--- |

| 版本 | BOM 版本号 |
| BOM 层级 | 产品 BOM / 半成品 BOM / 组件 BOM |
| 有效性 | 是否有效 |
| 定版标识 | 是否已定版（已定版后用于工艺与 MRP） |
| 状态 | 正常 / 禁用 |

**操作流程**：

1. 「新增」BOM，选择对应产品物料、设置 BOM 层级与版本。
2. 进入 BOM 明细，逐行添加物料（零件/组件），组件行可关联其子 BOM。
3. 点击行内 **查看层级** 以树形查看 BOM 的完整多层结构。
4. BOM 经定版（已定版）后，方可作为工艺规划与物料需求计算的基准。

# 第 7 章 工艺管理中心

工艺管理中心解决"怎么做"的问题：定义工序、编排工艺路线、绑定到 BOM 形成产品工艺规划、编制工艺内容，并提供查询。

## 7.1 工序信息定义

**路径**：工艺管理中心 → 工序信息定义（`/technology/oper/list-ui`）

![工序信息定义](images/40-oper.png)

**功能用途**：定义标准工序库，每道工序归属于某个加工单元，并设定标准工时、制造周期、是否生成生产计划等属性，供工艺路线/工艺流程引用。

**字段说明**：

| 字段 | 说明 |
| :--- | :--- |
| 工序编号 / 名称 | 工序唯一编码与名称（如 GX000002 主板装配作业工序） |
| 加工单元名称 / 类型 | 工序所属加工单元及其类型（人员/设备作业单元） |
| 工序工时(h) | 单件标准作业工时 |
| 制造周期(h) | 工序制造周期 |
| 是否生成生产计划 | 该工序是否参与生产计划拆分 |

**操作流程**：「新增」工序，选择所属加工单元，填写工时、周期、是否生成生产计划等并保存。

## 7.2 工艺路线管理

**路径**：工艺管理中心 → 工艺路线管理（`/basedata/flow/process/list-ui`）

![工艺路线管理](images/41-flow-process.png)

**功能用途**：定义产品的工艺路线（流程），即工序的时序串联（如：主板装配 → 机箱组装 → 主机装配 → 整机测试 → 包装入库）。一条流程描述了产品从开始到完工经过的工序顺序。

**操作流程**：「添加」流程，录入流程编码（如 `FLOW_PCHOST`）、流程描述，并编排工序的先后顺序；列表"流程时序过程"列直观展示工序串联关系。支持批量删除。

## 7.3 工艺流程管理

**路径**：工艺管理中心 → 工艺流程管理（`/technology/process-route/tree-ui`）

![工艺流程管理](images/42-process-route.png)

**功能用途**：将工艺路线**落到具体 BOM** 上，形成"产品工艺规划"。以 BOM 的层级树为骨架，为每个工艺节点绑定工序、加工单元、工时、周期、是否生成生产计划等，并支持**初始化**与**锁定定版**。

**界面说明**：顶部为工具条（选择 BOM、初始化工艺流程、锁定产品工艺规划），下方以树表展示该 BOM 的工艺节点层级。锁定后徽标显示"已锁定"，不可再编辑。

**字段说明**：工艺节点名称、工序编号、绑定工序、加工单元、加工单元类型、工时(h)、周期(h)、生成生产计划、排序号。

**操作流程**：

1. 点击 **选择BOM**，从已定版的 BOM 中选择一条（如 BOM-PC-HOST 台式电脑主机）。
2. 点击 **初始化工艺流程**（将按 BOM 结构生成工艺节点草稿，会覆盖未锁定的旧记录）。
3. 在树表中逐节点点击 **编辑工艺规划**，绑定工序、加工单元、工时等。
4. 各节点编制完成后，点击 **锁定产品工艺规划** 定版。
5. **锁定后**，该产品工艺及其工序不可再新增/编辑/删除，节点操作列变为"工艺内容编制 / 查看工艺"。

## 7.4 工艺内容编制

**路径**：工艺管理中心 → 工艺内容编制（`/technology/process-content/tree-ui`）

![工艺内容编制](images/43-process-content.png)

**功能用途**：在工艺规划锁定的基础上，为每个工艺节点编制详细的**工艺内容**（作业指导、参数、要点等），并标记编制状态（已完成）。是现场作业的执行依据。

**操作流程**：

1. 点击 **选择BOM**，载入该 BOM 的工艺节点树。
2. 对每个节点点击进入工艺内容编制向导（`wizard`），录入作业内容。
3. 节点"编制状态"显示"已完成 ✓"表示内容已编制完毕。

## 7.5 产品工艺查询

**路径**：工艺管理中心 → 产品工艺查询（`/technology/process-query/tree-ui`）

![产品工艺查询](images/44-process-query.png)

**功能用途**：以**只读**方式按 BOM 查看产品的完整工艺规划与工艺内容，供工艺评审、现场查阅、追溯使用，不提供编辑操作。

**操作流程**：选择 BOM → 展开工艺节点树 → 点击行内 **查看详情** 查看该节点的工序与工艺内容详情。

# 第 8 章 生产计划中心

生产计划中心是 MES 的执行调度核心，覆盖从订单录入、MRP 运算、两级派工、物料配套到计划下发、工单生成的完整链路。

## 8.1 生产订单录入

**路径**：生产计划中心 → 生产订单录入（`/production-order/plan/list-ui`）

![生产订单录入](images/70-po-plan.png)

**功能用途**（页面副标题）：承接客户需求、预测计划或 ERP 订单，锁定 BOM、产品物料、数量与交期，形成 MES 可排产的生产计划源头。订单提交审核通过后，可进行设备与员工作业派工，并在物料配套出库完成后进入生产计划下发。

**界面说明**：顶部为 KPI 卡片（生产订单总数 / 需求订单 / 预测订单 / 审核中 / 待下发 / 已下发）；列表展示订单及其审核状态、MRP 状态、下发状态、来源方式。

**字段/状态说明**：

| 字段 | 说明 |
| :--- | :--- |
| 订单编号 | 系统生成（如 DD20260612001） |
| 类型 | 需求订单 / 预测订单 |
| BOM 信息 / 产品物料 | 关联的产品 BOM 与物料 |
| 需求数量 / 计划交付日期 | 订单数量与交期 |
| 审核状态 | 未审核 → 审核中 → 已完成 |
| MRP 状态 | 未运算 → 已运算 / 已完成（含明细项数） |
| 下发状态 | 未下发 / 已下发 |
| 来源方式 | MANUAL（手工）等 |

**操作流程**：

1. 点击 **新增需求订单**（或 **新增预测订单**），选择产品 BOM、填写数量、计划交付日期等并保存（初始为"未审核/未下发"）。
2. 在订单行点击 **提交**，进入审批流程（订单审批，见第 12 章）。
3. 审核通过后点击 **运算** 触发 MRP 物料需求运算，生成需求明细（MRP 状态变为已完成，记录运算时间）。
4. 订单达到"审核通过 + 派工完成 + 物料配套出库完成"后，进入"生产计划下发"。
5. 列表支持 **导出**；可按关键字、订单类型、审核状态、下发状态筛选。

3. 第③步审核/补全工序与工艺。
4. 第④步建立工单并完成排人，形成可执行的生产准备。

# 第 16 章 典型业务全流程演练

本章以演示数据"**台式电脑主机（PC_HOST）**"为主线，串联各模块，帮助新用户走通一条完整业务闭环。建议按序操作。

> 该产品由"台式电脑主机 → 台式电脑主机半成品 → 主板单元 / 机箱单元"逐层构成；工艺路线为：主板装配 → 机箱组装 → 主机装配 → 整机测试 → 包装入库。

**阶段一：主数据准备（基础数据中心）**

1. **物料信息定义**：建立成品/半成品/组件/零件物料（PC_HOST、CPU、MEMORY、PCB_BOARD、POWER_SUPPLY、CASE_SHELL 等），设置计量单位、安全库存、提前期、来源。
2. **班组员工定义**：建立 TEAM01 主板装配班组、TEAM02 机箱组装班组、TEAM03 总装测试班组并加入员工。
3. **加工单元定义**：建立"电脑组装单元/整机测试单元/包装入库单元"，绑定执行班组。
4. **设备 / 编组设备定义**：建立设备并按需编组。
5. **库房库位定义 + 库存管理**：建立 KF001 配件库、CP001 产品库及库位，初始化原材料库存。

**阶段二：产品与工艺设计（产品数据中心 + 工艺管理中心）**

6. **零部件定义 / 产品 BOM 管理**：建立三层 BOM（产品/半成品/组件），并**定版**。
7. **工序信息定义**：建立主板装配、机箱组装、主机装配、整机测试、包装入库等工序，归属加工单元。
8. **工艺路线管理**：编排工序时序（FLOW_PCHOST）。
9. **工艺流程管理**：选择已定版 BOM → 初始化工艺流程 → 逐节点绑定工序 → **锁定产品工艺规划**。
10. **工艺内容编制**：为各工艺节点编制作业内容至"已完成"。

**阶段三：计划与派工（生产计划中心 + 流程配置工具）**

11. **生产订单录入**：新增需求订单（产品 PC_HOST、数量、交期）。
12. **提交审核**：订单提交后由"流程办理"按订单审批流程审批通过。
13. **MRP 运算**：对已审核订单执行"运算"，生成物料需求明细。
14. **设备作业派工 / 员工作业派工**：为各工序绑定设备、（负载均衡）分配员工。

**阶段四：物料配套（生产计划中心 + 库房管理中心）**

15. **物料需求计划（明细）**：对净需求行 **申请** 并 **生成配套出库单**。
16. **配套出库确认**：库房按整单校验库存后确认出库，扣减库存并生成出库流水。

**阶段五：下发与执行（生产计划中心 + 计划管理 + 在制品管理）**

17. **生产计划下发**：对"待下发"计划 **拆分任务** 并下发，生成生产工单。
18. **生产工单查询 / 工单管理**：跟踪工单审批、动工、完工、交付状态（甘特图）。
19. **SN 通用过程采集**：现场按工艺路线逐站采集每个 SN 的 OK/NG 结果。
20. **完工交付**：工单完工交付后进入"已交付工单"，完成闭环。

**阶段六：成品入库与可视化（库房管理中心 + 数字化平台）**

21. **手工/计划入库确认**：成品按需入库登账并生成入库流水。
22. **库存明细 / 出入库流水查询**：核对库存与流水，全程可追溯。
23. **数字化平台 / 智能助手**：通过数据中心大屏与智能数据助手查看产量、合格率、库存等指标。

## 17.3 常见问题（FAQ）

**Q1：登录提示"请输入正确的验证码"？**
A：验证码为 4 位数字、区分本次会话；点击验证码图片可刷新后重新输入。

**Q2：工艺流程页面提示"请先选择已锁定的 BOM"，看不到工艺节点？**
A：工艺流程/内容/查询均需先点击"选择 BOM"载入；其中工艺内容编制与查询要求 BOM 已完成工艺规划锁定。

**Q3：生产订单无法下发？**
A：计划下发只展示"工单已审批 + 设备与员工派工均完成 + 等待下发"的计划，请先完成审批与两级派工，并完成物料配套出库。

**Q4：配套出库确认失败、库存未变化？**
A：配套出库按整单校验库存，存在任一物料库存不足时整单不出库、不扣减任何库存，请先补足库存或调整需求。

**Q5：删除了数据还能找回吗？**
A：系统为软删除（`is_deleted=1`），数据仍在库中，可由管理员通过数据库恢复；列表默认不再展示。

**Q6：菜单更新后用户看不到新功能？**
A：菜单需授权给对应角色，且用户需重新登录刷新权限缓存。
