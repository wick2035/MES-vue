# MES 开发规范

> 本规范用于约束本项目的二次开发，**重点是保持 UI / 设计一致性**——新人最容易犯的错是硬编码样式、引入另一套 UI 体系或不遵循既有模板范式，导致界面风格走样。请在动手前通读本文。
>
> 配套文档：[README.md](../README.md)（从 0 部署）、[CLAUDE.md](../CLAUDE.md)（构建运行与模块范式速查）、[docs/spTable.md](spTable.md)、[docs/spLayer.md](spLayer.md)。

---

## 0. 总则（红线）

1. **不引入第二套 UI 体系**：本项目前端只用 `layui` + 自封装 `sp*` 组件 + FreeMarker。**禁止**引入 Vue / React / Element / Ant Design / Bootstrap 等任何其它 UI 框架或皮肤。
2. **不硬编码样式值**：颜色、圆角、阴影、字体一律使用 `theme.css` 的设计令牌 `var(--sp-*)`。**禁止**写死 `#2563EB`、`#f2f2f2`、`8px` 这类裸值。
3. **复用既有语义类与组件**，不要自造重复实现（见第 2、3 节）。
4. **照现有代码对齐**：新增模块前先找一个最接近的现有模块（如 `basedata` 下的物料 / 班组）照抄结构。
5. **中文优先**：提交信息、注释、UI 文案以中文为主，跟随现有风格。
6. **git 不出现任何 AI / Claude 字样**（提交信息、文件内容均不得包含）。

## 1. UI / 设计规范（核心）

### 1.1 设计语言

浅色精密工业风（Industrial Light）：工业蓝 `#2563EB` 主色 · 圆角 4–6px · 1px 发丝线 · 阴影克制 · 间距紧凑 · 系统字体。

### 1.3 现成语义类（复用，勿重复造）

| 类名 | 用途 |
| --- | --- |
| `.splayui-container` > `.splayui-main` | 页面外壳卡片 + 内边距 |
| `.sp-section-title` | 分段标题（左侧蓝色竖条 + 加粗），替代散落的标题样式 |
| `.sp-required`（加在 `label` 上） | 必填项，自动追加红色 `*` |
| `.sp-readonly`（加在 `input.layui-input` 上） | 只读输入框（浅灰底 + 次级文本色） |

### 1.4 尺寸规范

- 按钮高度三档：默认 `36px`、`.layui-btn-sm` `30px`、`.layui-btn-xs` `24px`；按钮间距 `8px`。
- 输入框高度 `36px`；表单 label 右对齐，宽度 120–150px。
- 表格：表头底色 `var(--sp-surface-2)`，行 hover `var(--sp-primary-tint)`，单元格纵向 padding 约 `9px`，边框 1px。

### 1.5 Do / Don't

| ✅ Do | ❌ Don't |
| --- | --- |
| 颜色 / 圆角 / 阴影用 `var(--sp-*)` | 硬编码 `#2563EB`、`#f2f2f2`、裸 `px` 色值 |
| 复用 `.sp-section-title` / `.sp-readonly` / `.sp-required` / `.sp-badge-*` | 自写行内样式重复实现这些效果 |
| 通过 `common/common.ftl` 引入 `theme.css` | 单独引第三方 UI 框架 / 皮肤 / CDN 主题 |
| 用 layui + `sp*` 组件渲染表格 / 弹窗 | 引入 Vue / React / Element 等另一套体系 |
| 图标用 Font Awesome 4.7 或 layui icon | 混入其它图标库 |
| 按钮 / 输入框沿用既有尺寸档位 | 自定义不一致的高度 / 间距 |
| 新页面复制最接近的现有模板再改 | 从零手写偏离范式的页面结构 |

## 2. 前端组件与模板范式

### 2.1 资源引入顺序（固定，见 common.ftl）

CSS：`layui.css` → `font-awesome` → `public.css` → `theme.css`
JS：`jquery` → `layui.js` → `spConfig.js` → `spUtil.js` → `layuimodule/config.js`

`spTable` / `spLayer` 等为 layui 自定义模块，在页面里通过 `layui.use([...])` 按需加载，**不要**在 `common.ftl` 之外乱改引入顺序。

### 2.2 sp 自封装组件 API

```javascript
// 列表表格：POST 取数，parseData 读 res.data.records / res.data.total
spTable.render({
  elem: '#js-xxx-table',
  toolbar: '#js-xxx-toolbar-top',     // 顶部批量按钮模板
  url: '${request.contextPath}/{域}/{模块}/query',
  method: 'POST',
  cols: [[ /* ... */ ]],
  page: true,
  limits: [10, 20, 50, 100],
  limit: 10
});

// 弹窗编辑：iframe 打开 add-or-update-ui，回调刷新
spLayer.open({
  type: 2,
  content: '${request.contextPath}/{域}/{模块}/add-or-update-ui',
  spWhere: { id: id },                // 传参
  spCallback: function (res) { /* 成功后回调，通常 table.reload */ }
});

// 工具方法
spUtil.submitForm({ url, data, success });  // 表单提交
spUtil.ajax({ url, type, data, success });   // 通用 ajax
```

子页 → 父页回传（选择弹窗）约定：子页设置 `window.spChildFrameResult = { code: 0, data: [...] }`，父页在 `spCallback(res)` 接收。

## 3. 后端 CRUD 范式

新增模块务必照现有代码对齐（参考 `basedata` 下既有模块）。

- **实体**：继承 `common.BaseEntity`（`id` 雪花串、`createTime/createUsername/updateTime/updateUsername` 自动填充）；`@TableName("sp_xxx")`；软删字段 `@TableField("is_deleted") private String deleted;`；仅展示用的联表字段加 `@TableField(exist = false)`。
- **软删约定**：`is_deleted` 取值 `0` 正常 / `1` 删除 / `2` 禁用。查询统一 `ne("is_deleted","1")` 或 `eq("is_deleted","0")`，**不物理删除**。
- **Controller**：`@Controller` + 继承 `common.BaseController`；`@RequestMapping("/{域}/{模块}")`；`*-ui` 的 `@GetMapping` 返回模板路径字符串；数据接口 `@PostMapping + @ResponseBody` 返回 `common.Result`（`Result.success(data)` / `Result.failure(msg)` / `Result.success(data, msg)`）。
- **分页**：请求对象继承 `common.BasePageReq`（`current`/`size`/`orderBy`）；返回 MyBatis-Plus `IPage`；前端 `spTable` 的 `parseData` 读 `res.data.records` / `res.data.total`。
- **Service / Mapper**：`IService` / `ServiceImpl` + `BaseMapper`；自定义联表写在 Mapper 接口 + 对应 XML（`mapper/{域}/XxxMapper.xml`）。
- **唯一性校验**：Service 实现 `isXxxCodeDuplicate(code, excludeId)`（`ne is_deleted '1'`，编辑排除自身），保存前在 Controller 调用。
- **密码**：哈希算法统一为 `new Md5Hash(password, username, 3)`（盐 = 用户名，迭代 3 次），新增 / 重置 / 改密三处保持一致。
- **菜单**：存于 `sp_sys_menu`，通过 SQL 脚本 `INSERT IGNORE` 注册，并向 `sp_sys_role_menu` 授权（管理员 role code `888888`）。

## 4. 目录与命名约定

- Java 源码根包 `com.wangziyang.mes`，按业务域分包：`basedata`（基础数据）、`technology`（工艺）、`system`（用户 / 角色 / 菜单 / 部门）等。域内：`controller` / `entity` / `mapper` / `request` / `service` / `service.impl`。
- MyBatis XML：`mes/src/main/resources/mapper/{域}/XxxMapper.xml`。
- FreeMarker 模板：`mes/src/main/resources/templates/{域}/{模块}/list.ftl | addOrUpdate.ftl | select.ftl`。
- 静态资源：`mes/src/main/resources/static/`（自封装组件在 `js/layuimodule/sp/`）。
- 数据表统一 `sp_` 前缀。

## 5. SQL 脚本规范

- 脚本放 `scripts/sql/`，命名 `{feature}-upgrade-YYYYMMDD.sql`，**手动执行**（无 Flyway / Liquibase）。
- 必须**幂等、可重复执行**：建表用 `CREATE TABLE IF NOT EXISTS` / `DROP TABLE IF EXISTS`；加列前用 `INFORMATION_SCHEMA` 判断列是否存在（或 `PREPARE/EXECUTE` 动态执行）；数据用 `INSERT IGNORE` 或 `INSERT ... WHERE NOT EXISTS`。
- 脚本头部加 `SET NAMES utf8mb4;`；字符集 / 排序规则统一 `utf8mb4` / `utf8mb4_0900_ai_ci`。
- 菜单变更：`INSERT IGNORE` 写 `sp_sys_menu`，并向 `sp_sys_role_menu` 给管理员（role code `888888`）授权。
- 新脚本写好后，按需同步进 `scripts/sql/init-all.sql`（全量初始化脚本，仅供从 0 部署；做法见 [README 第六节](../README.md)）。
- 执行命令务必带 `--default-character-set=utf8mb4`，否则中文乱码。

## 6. 构建、运行与提交

- 构建 / 运行**必须**带 `-s .\.codex-maven-settings.xml`（阿里云镜像 + 项目内 `.m2/repository`）：

  ```powershell
  mvn -s .\.codex-maven-settings.xml -f .\mes\pom.xml -DskipTests compile
  mvn -s .\.codex-maven-settings.xml -f .\mes\pom.xml -DskipTests spring-boot:run
  ```

- 改完代码用上面的 `compile` 命令编译验证（IDE 启动亦可）。
- 数据库结构变更：新增 `scripts/sql/*-upgrade-*.sql` 并执行，**不要**直接改库结构而不留脚本。
- 提交信息、注释、UI 文案以中文为主；**git 中不得出现任何 AI / Claude 相关字样**。
